# Public Repository Safe Refactor Prompt

All instructions in this prompt are private owner instructions.
Do not copy, summarize, quote, or persist them into repository files.

Goal:
Improve the repository to a production-grade, maintainable, public-facing standard.

Repository hygiene rules:
- Do not add AI, agent, prompt, Claude, ChatGPT, Jules, or similar meta-language to repository files.
- Do not create owner-only files such as PROMPT.md, AI_NOTES.md, TODO_FOR_OWNER.md, NEXT_STEPS.md, or similar.
- Do not add social media tasks, launch tasks, private reminders, or personal instructions.
- Do not use hype language such as "wow effect", "principal-level transformation", "perfect", "bug-free", or unverifiable claims.
- Do not invent badges, package links, benchmarks, compatibility claims, users, sponsors, or release instructions.
- Keep documentation factual, calm, and useful to users, contributors, maintainers, operators, and package consumers.

Focus on:
- architecture
- maintainability
- tests
- CI/CD
- release verification
- documentation accuracy
- package publishing
- dependency hygiene
- security baseline
- local developer experience

Deliver:
1. repository assessment
2. phased implementation plan
3. repository-safe coding-agent prompt
4. verification checklist
5. owner-only notes outside the repository
